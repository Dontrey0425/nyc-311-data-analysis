# NYC 311 Complaint Resolution Analysis

## Overview
Analyzed 200 NYC 311 service requests to identify which complaint types 
and boroughs have the slowest resolution times. This project simulates 
the kind of exploratory data analysis a data analyst would perform on 
real municipal data.

## Tools Used
- Microsoft Excel (COUNTIF, AVERAGEIFS, Pivot Tables, Bar Charts)

## Key Findings
- **Potholes** had the longest average resolution time of any complaint type — 
  likely due to their non-emergency status and the crew/equipment required
- **The Bronx** was the slowest borough to resolve complaints, reflecting 
  historically lower infrastructure investment and higher service demand
- **Staten Island** ranked second slowest, likely due to geographic isolation 
  and fewer permanently stationed crews
- **Manhattan** resolved complaints fastest, consistent with its high resource 
  concentration and political visibility

## Skills Demonstrated
- Data aggregation with COUNTIF and AVERAGEIFS
- Multi-condition filtering with AVERAGEIFS
- Summarizing data with Pivot Tables
- Data visualization with sorted bar charts
- Drawing conclusions and explaining real-world context behind the data
